By default Uxaria uses a 12-column grid system. Grid dimensions are in em, making them scale proportionally with the base font size. The default grid gutter is 1.4em - equal to the line height by design.

{% endblock lead %} {% block body %}

Grid & Responsive Design

Instead of shrinking / widening column and gutter widths in responsive design (like Bootstrap and most other frameworks do), Uxaria reduces the number of colums available. On tablet in portrait orientation each row consists of 9 grid columns. This is further reduced to 6 grid colmns on mobile phones in landscape orientation. For mobile phones in portrait orientation we finally stack the columns vertically.

The motivation behind this design decision is to give you as a web designer fine-grained control over responsive layout mechanics while keeping all proportions within content elements as constant as possible on all devices. Assign the helper classes span*-phone and span*-tablet to rearrange elements for smaller screens.

Regular Grid

The regular grid system is very close to the common 960 grid system on screens with a device width of 1024px. It's proportionally wider if the browser window or device is wider.

Tiles Grid

This grid reduces the gutter between the blocks to just a few pixels. It enables you to create tiles as popularised by Windows 8 Modern UI.

To use this grid, simply add the additional class tiles to the row.

Alternative Grids

Add an additional class to change the whitespace of a grid row:

{{ docs.codeblock('<div class="row alternative"> ... </div>', 'html', false) }}

or:

{{ docs.codeblock('<div class="row tiles"> ... </div>', 'html', false) }}

Responsive Helper Classes

To control the layout of narrower grid systems with 6 or 9 columns for phones and tablets, use span*-phone, span*-tablet, offset*-phone and offset*-tablet helper classes:

{{ docs.codeblock('<div class="row"> <div class="span3 span2-phone">...</div> <div class="span9 span6-tablet span4-phone">...</div> </div>', 'html', false) }}

To undo an offset, add offset0-phone or offset0-tablet.
